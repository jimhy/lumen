## 二、我替你拍板的 UX 决策（明天看着不顺手就告诉我改）

- **A 双击目录** = 控制端本地展开/折叠，**不 cd**（cd 会污染被控端正在跑的 shell；展开纯本地体现 #6）。
- **B 复制/粘贴入口** = 右键菜单。远程树行右键「复制」/ 目录「粘贴到此目录」；本地树同。
  - **下载流程**：远程视图右键文件/夹「复制」→ 切到**本地视图**右键某目录「粘贴到此目录」。
  - **上传流程**：本地视图右键文件/夹「复制」→ 切到**远程视图**右键被控端某目录「粘贴到此目录」。
  - （剪贴板跨视图保留；会话结束自动清掉「远程来源」的剪贴板防野路径。）
- **C 覆盖提示** = 粘贴时**一次性决策**模态：`[覆盖全部] [跳过已存在] [取消]`，套用整次递归（不逐文件弹）。
  上传方向在被控端探到首个同名时弹；下载方向在控制端本地探到顶层同名时弹。
- **D** 传输分块 64 KiB + ACK 滑动窗口背压（窗口 4）；单文件 Fetch 上限 2 GiB（可调）。
- **F 读边界** = 信任已配对控制端（可读被控端任意路径，仅控制字符拒 + 大小写上限）。
- **F 写边界（安全）** = 被控端写入做**路径穿越硬防御**（拒 `..`/分隔符/盘符/保留设备名 + canonical 子树断言），
  但**目标目录本身不限作用域**（控制端是已配对鉴权方、本可在被控端跑任意命令，全盘可写与现有威胁模型一致）。
  → 若你要更严（限在被控端当前浏览根的子树内），告诉我，几行可加。
- **H** 远程树工具条有「显示隐藏项」勾选（默认关），勾上才传/显示 `.env` 等点文件。
- **G** remote→remote / local→local 复制粘贴本期**不做**（菜单按侧禁用）。

## 四、已知遗留（不影响验收，待后续硬化；挂 task）

- **被控端文件服务无界化**（review MED-1，原 part3c-1 task#8）：被控端每个 ListDir/Fetch/Put 请求起一条
  OS 线程、无全局并发上限。控制端侧已对**下载递归 ListDir** 和**文件 Fetch / 上传 Put** 加了并发上限（各 4~6），
  实际负载被控制端钳住；但被控端服务本身未有界化。后续统一为「常驻 service 线程 + 有界队列」。
- **上传停滞清扫**（review 片5 MED-1）：下载方向有 30s 停滞超时清理，上传方向暂无——靠 WS 断线 + 会话结束兜底
  （H1/H2 已修后，正常不会挂死）。
- **非 UTF-8 路径**（review 片2 MED-2）：协议 path 经 `display()` 有损，被控端含非 UTF-8 字节的路径不可展开。
  本项目被控端为 Windows、几乎全 UTF-8，MVP 接受；若要严谨改 `Vec<u8>` 不透明键。
